## Часть 1 — Flow Efficiency (новая метрика)

### Определение

```
Flow Efficiency = (avg Cycle Time / avg Lead Time) × 100%
```

**Что измеряет:** какой процент от общего времени жизни задачи она реально находилась в работе. Оставшееся время — ожидание в очереди, блокеры, handoff'ы.

**Примеры:**
- Lead Time 10d, Cycle Time 3d → Flow Efficiency = **30%** (70% задача ждала)
- Lead Time 5d, Cycle Time 4d → Flow Efficiency = **80%** (отличный процесс)

**Бенчмарки (Flow Metrics, "Making Work Visible"):**

| Значение | Интерпретация |
|---------|--------------|
| > 40% | World-class |
| 15–40% | Норма |
| < 15% | Много блокеров / очередей |

Примечание: здесь `good >= 40` и `bad <= 15`, поэтому `setKpiStatus` нужно вызвать с инвертированной логикой или добавить параметр `higherIsBetter`:

```js
// Расширить сигнатуру setKpiStatus:
function setKpiStatus(cardId, value, goodThreshold, badThreshold, higherIsBetter = true) {
  const el = document.getElementById(cardId);
  el.classList.remove("kpi-status-good", "kpi-status-bad");
  if (higherIsBetter) {
    if (value >= goodThreshold) el.classList.add("kpi-status-good");
    else if (value <= badThreshold) el.classList.add("kpi-status-bad");
  } else {
    if (value <= goodThreshold) el.classList.add("kpi-status-good");
    else if (value >= badThreshold) el.classList.add("kpi-status-bad");
  }
}

// Вызовы (обновить все в updateDashboard):
setKpiStatus("kpi-cycle-time-card", d.cycleTimeDays,        5,  10,  false); // lower is better
setKpiStatus("kpi-lead-time-card",  d.leadTimeDays,         10, 20,  false); // lower is better (или timeToMarket после п.2)
setKpiStatus("kpi-throughput-card", d.throughput > 0, true, false);          // просто toggle как было
setKpiStatus("kpi-flow-eff-card",   d.flowEfficiencyPercent, 40, 15, true);  // higher is better
```
