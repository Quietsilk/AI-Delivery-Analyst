## Часть 2 — Баги в расчётах

### BUG-1 🔴 Critical: Done-задачи без resolutiondate не считаются выполненными

**Файл:** `server.py`, функция `fetch_jira`, строки 140–144

**Суть:** Changelog фетчится только для задач у которых есть `resolutiondate` или текущий статус в `STARTED`. Если Jira-проект настроен **без resolution field** (распространённая конфигурация) — `resolutiondate = None`. Задача в статусе "Done" не попадает в `needs_changelog`, changelog не фетчится, `last_done = None`, `resolved_at = None` → задача не считается выполненной. **Throughput = 0** при таких проектах.

**Как исправить:**
```python
needs_changelog = [
    i["key"] for i in all_issues
    if i.get("fields", {}).get("resolutiondate")
    or i.get("fields", {}).get("status", {}).get("name", "").lower() in STARTED
    or i.get("fields", {}).get("status", {}).get("name", "").lower() in DONE  # ← добавить
]
```

---

### BUG-2 🟠 Medium: Cycle Time считается от первого старта, не от последнего

**Файл:** `server.py`, строка 44

**Суть:**
```python
started = next((t for t in transitions if t["to"].lower() in STARTED), None)
```
Берёт **первый** переход в STARTED. Если задача шла `In Progress → Backlog → In Progress → Done` — цикл считается с самого первого старта. Cycle Time раздувается у задач, которые возвращали в бэклог.

**Как исправить:** взять **последний** переход в STARTED перед `resolved_at`:
```python
started = next((t for t in reversed(transitions)
                if t["to"].lower() in STARTED
                and (not last_done or t["date"] <= last_done["date"])), None)
```

---

### BUG-3 🟠 Medium: Reopened не фильтруется по периоду

**Файл:** `server.py`, строка 95

**Суть:** `throughput` фильтруется по cutoff-периоду. `reopenedCount` — нет. Считает все переоткрытия за всё время, даже если выбран период 7d. Несогласованность.

**Как исправить:** фильтровать reopened по `resolved_at` или дате перехода в рамках периода. Как минимум считать только для `completed` (задач в периоде):
```python
"reopenedCount": sum(1 for i in completed if i["reopened"]),
```

---

### BUG-4 🟠 Medium: Задачи возвращённые из In Progress в Backlog невидимы

**Файл:** `server.py`, функция `fetch_jira`, строки 140–144

**Суть:** Задача прошла `In Progress → Backlog`. Текущий статус не в STARTED, resolutiondate нет → changelog не фетчится → `started_at = None` → попадает в Backlog как будто никогда не стартовала. Не видна в `in_progress`, Backlog Size завышен.

**Как исправить:** фетчить changelog для всех задач у которых текущий статус не является ни STARTED ни DONE — т.е. добавить проверку на "был ли ранее в работе" через базовый fetch. Либо фетчить changelog для всех задач (компромисс по производительности).

---

### BUG-5 🟡 Low: throughput и completedCount — дубль

**Файл:** `server.py`, строки 90 и 94

**Суть:** `throughput = len(completed)` и `completedCount = len(completed)` — одно и то же значение в ответе. Вводит путаницу: зачем два поля с одним значением?

**Как исправить:** удалить `completedCount` из ответа, оставить `throughput`. Или дать им разный смысл: `completedCount = len(all_completed)` (все закрытые за все время), `throughput = len(completed)` (за период).
