## Часть 1 — Блокеры (обязательно до публикации)

### 1.1 Убрать метрику Done Rate полностью

**Проблема:** `doneRatePercent = len(completed_in_period) / len(all_jql_issues) * 100`

Числитель фильтруется по периоду, знаменатель — нет. Знаменатель растёт с историей проекта, что делает метрику несравнимой между периодами и командами. Сломанная метрика в аналитическом инструменте хуже чем её отсутствие.

**Что сделать:**

**`server.py` — `calculate_metrics()`:**
- Удалить строку `"doneRatePercent": round(...)` из возвращаемого словаря

**`server.py` — `call_openai()`:**
- Удалить строку `f"- Done Rate: {m['doneRatePercent']}%\n"` из промпта

**`server.py` — `_handle()`:**
- Строка `icon = "🟢" if metrics["doneRatePercent"] >= 80 else ...` — заменить логику иконки:
  ```python
  # Иконка по Throughput: зелёный если > 0, жёлтый если > 0 но есть reopened, красный если 0
  icon = "🔴" if metrics["throughput"] == 0 else ("🟡" if metrics["reopenedCount"] > 0 else "🟢")
  ```
- Удалить строку `f"{icon} Done Rate: {metrics['doneRatePercent']}%"` из `tg_text`
- Удалить поле `"doneRatePercent"` из возвращаемого JSON-ответа

**`server.py` — строка 6 (docstring):**
- Убрать `doneRatePercent` из описания Returns

**`ai-delivery-analyst-dashboard.html` — KPI row:**
- Удалить карточку `id="kpi-done-rate-card"` (строки 542–547)
- Изменить grid: `grid-template-columns: repeat(4, minmax(0,1fr))` (было 5)
- Адаптировать responsive breakpoints: `@media (max-width: 900px)` → `repeat(2,1fr)` остаётся, `@media (max-width: 520px)` → `1fr` остаётся — без изменений

**`ai-delivery-analyst-dashboard.html` — JS:**
- Удалить из массива flash: `"kpi-done-rate-card"` (строка 987)
- Удалить строку `document.getElementById("kpi-done-rate-value").textContent = ...` (строка 1018)

**Файл:** `.github/workflows/tests.yml` (новый файл)

**Содержимое:**
```yaml
name: Tests

on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: "3.11"
      - run: python3 -m unittest discover -s tests -v
```

Никаких новых зависимостей — проект zero-dependency, этот workflow это подтверждает.
